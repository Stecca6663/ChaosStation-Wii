# Chaos Station

A **New Super Mario Bros. Wii** hack: custom worlds, layouts, and a full **streamed soundtrack** replacement, loaded through **Riivolution**.

### What makes this mod unusual

**Almost everything here was produced through an AI-assisted workflow.** Design choices, refactors, tooling, and iteration were driven by **large language models and coding agents**—not by a human using a graphical interface. **I (Stecca)** own the project and **review, playtest, and sign off** but that's all. The experiment I wanted to do is, after all, how much an AI can do on their own.

**The levels themselves are generated by Python.** The geography, object placement, and bulk of course data come from scripts in this repo (notably the **`create_mod.py`** family and related generators), which encode the design the AI helped shape. That means Chaos Station is part **procedural / data-driven hack**, part **soundtrack and systems overhaul**—with the same AI+script loop applied to music tooling (BRSTM/BRSAR pipelines), layout notes, and helper utilities you see in the tree.

> **Region:** US / `SMN` (see `output/riivolution/nsmbw_mod.xml`). Other regions may need XML or file tweaks.

---

## What’s in the mod

- **Custom levels** under `Stage` (`.arc` level files), built from **Python-generated level data** rather than hand-placed in a GUI for every tile.
- **Objects, Layout, and UI** patches (pause menu, title flow, staff roll, etc.).
- **Custom music** via `Sound/stream/*.brstm` and a patched **`wii_mj2d_sound.brsar`** (sizes, hurry variants, channel layout fixes)—with maintenance scripts also evolved in the same AI-assisted workflow.

The repository holds **scripts and sources** used to build the pack. The assembled **`ChaosStation`** folder and **`output/`** tree are **not** committed; but you can download them in the release section.

## Playing the mod

1. Own a **legitimate copy** of New Super Mario Bros. Wii.
2. Install **Riivolution** (Wii) or use **Dolphin** with Riivolution support.
3. Copy the built pack to the SD card (Wii) or load paths (Dolphin) so this structure exists:

   ```text
   /Riivolution/ChaosStation/     ← mod files (Stage, Object, Layout, Sound, …)
   /Riivolution/nsmbw_mod.xml       ← or your chosen XML path
   ```

   Exact paths depend on your Riivolution config; the XML in this repo maps **`/ChaosStation/...`** onto disc paths such as `/Stage`, `/Sound/stream`, and **`/Sound/wii_mj2d_sound.brsar`**.

4. Enable the patch (e.g. **“Chaos Station v2” → Custom Levels → Enabled**).

## Building / maintaining audio (developers)

Paths assume a Windows-style checkout; adjust if needed.

| Task | Command / script |
|------|------------------|
| Patch **BRSAR** embedded sizes after changing any `.brstm` | `python patch_brsar.py` |
| Generate **hurry** (~1.15×) variants from normal streams | `python batch_create_fast_tracks.py` then `python patch_brsar.py` |
| Verify BRSAR size fields vs files on disk | `python verify_brsar_stream_sizes.py` |
| Compare mod streams vs vanilla (channels, rate, loops) | `python audit_stream_brstm.py` |
| Resample / fix **4ch** ground & cave streams (vanilla layout) | `fix_all_music.py` / `rebuild_track` workflow |
| Import a **44.1 kHz** (or other) BRSTM as **underwater** `BGM_SUICHU.32.brstm` | `python import_suichu_brstm.py "path\to\file.brstm"` then fast + patch as above |

**Tooling expectations**

- **Python 3**
- **VGAudioCli** as `VGAudioCli.exe` in the repo root (not committed; see `.gitignore` for `*.exe`)
- **FFmpeg** at `lac/tools/ffmpeg/ffmpeg.exe` (or edit scripts to match your install)
- Vanilla **`extracted files/Sound/wii_mj2d_sound.brsar`** for `patch_brsar.py` to copy from before patching

## Repository layout (high level)

| Path | Role |
|------|------|
| `output/riivolution/` | Riivolution XML and launcher-facing config |
| `output/ChaosStation/` | Game-relative mirror: Stage, Object, Layout, Sound, … |
| `create_mod.py` | Primary **level-generation** entrypoint (and related root scripts) |
| `patch_brsar.py` | Updates stream size table in `wii_mj2d_sound.brsar` |
| `fix_all_music.py` | BRSTM normalize (e.g. 32 kHz, 4ch where required) |
| `create_fast_track.py` / `batch_create_fast_tracks.py` | Hurry BGM generation |
| `tools/` | Level and data helpers |
